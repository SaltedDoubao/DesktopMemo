# 常见问题

## 1. 列表里显示的“修改日期”不准确 / 启动后被刷新了？

应用启动时可能会执行数据迁移或索引重建（例如将旧格式索引迁移到 SQLite）。在某些迁移流程中，可能会更新文件时间戳以保证索引一致性，从而导致“看起来像被刷新”的现象。

建议排查顺序：

1. 查看 `.memodata/.logs/` 中最近的日志是否出现 “Migration / 迁移” 相关记录
2. 若你是从旧版本升级而来，优先确认是否发生了索引迁移
3. 若问题持续存在，可在 Issues 中反馈并附上日志（注意脱敏）

## 2. UI/文字看不清（纯白背景、对比度过低）

建议先尝试：

- 调整系统主题（浅色/深色）
- 调整应用透明度
- 选择对比更高的桌面壁纸

如果你希望应用支持“自动对比度优化”，可以在 Issues 中提出场景与期望效果，便于评估实现方式。

## 3. 应用启动失败或异常崩溃怎么办？

通用处理步骤：

1. 先备份 `.memodata`
2. 检查 `.memodata/.logs/` 中的错误日志
3. 若是数据损坏导致无法启动，可尝试临时移走 `.memodata`（让应用重新生成）以确认问题是否与数据相关

## 4. 如何并排固定多个独立备忘录窗口？

这是一个常见诉求，目前属于规划/讨论方向。你可以在 Issues 中描述你的使用场景（例如多窗口独立置顶/独立位置记忆/独立透明度），便于拆解需求与评估工作量。
